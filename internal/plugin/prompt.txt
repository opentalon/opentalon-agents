Persistent agents: automations you author once in the Talon language, that are stored and run deterministically — no model in the loop at run time.

Use these actions to manage agents for the current group:
- create(name, talon_source, [description], [triggers]) — author a new agent. The Talon source is validated before it is stored; if it is invalid you get compile diagnostics back — fix the source and retry.
- list / show(id) — inspect agents (show returns the full Talon source).
- run(id) — execute an agent's program now and return the result.
- update(id, talon_source, [triggers]) — replace the program (re-validated).
- enable(id) / disable(id) / delete(id) — lifecycle.

Authoring notes:
- talon_source is Talon, the same language talon-plugin executes. Write a `workflow "..." { step "..." { mcp "<server>" "<tool>" { ... } } }` program; MCP steps run through the host, exactly as talon-plugin runs them.
- triggers is an optional JSON array, e.g. [{"type":"schedule","cron":"0 9 * * 1-5"}]. Trigger types: manual, schedule, poll, webhook. Triggers are stored now and become active in a later phase; for now, drive agents with run(id).
- Keep one clear task per agent. Validate by creating (invalid source is rejected) or by run(id).
